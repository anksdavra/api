Post List  ¶

Returns a list of articles based on parameters passed on.

Posts list
GET/posts{?ids,tags,tags_exclude,categories,categories_exclude,series,series_exclude,packages,packages_exclude,authors,authors_exclude,post_types,sticky,post_types,offset,page,per_page,orderby,order,posts_exclude}

Example URI

GET /posts?ids=1234,43221&tags=music&tags_exclude=food&categories=beauty&categories_exclude=books&series=loves&series_exclude=strong&packages=stylist&packages_exclude=trainning&authors=willbays&authors_exclude=Adrian.Cristea&post_types=post&sticky=1&post_types=post&offset=1&page=1&per_page=3&orderby=title&order=DESC&posts_exclude=12345,23456
URI Parameters
HideShow
ids
integer (optional) Example: 1234,43221
tags
string (optional) Example: music
tags_exclude
string (optional) Example: food
categories
string (optional) Example: beauty
categories_exclude
string (optional) Example: books
series
string (optional) Example: loves
series_exclude
string (optional) Example: strong
packages
string (required) Example: stylist

live (optional, string)

packages_exclude
string (required) Example: trainning

club (optional, string)

authors
string (optional) Example: willbays
post_types
string (optional) Example: post

Just for clarification, here is a valid list of post_types:

  • post

  • page

  • longform

  • quiz_post

  • sponsored_post

  • sponsored_longform

  • sponsored_quiz_post

posts_exclude
string (optional) Example: 12345,23456

Posts with the specified ids will be omitted from the results

authors_exclude
string (optional) Example: Adrian.Cristea

All string params accept multiple values separated by commas. Example: ?tags=music,food

sticky
integer (optional) Example: 1

Accepts 1 for true and 0 for false:

  • if 1, it will return ONLY sticky posts

  • if 0, it will return NO sticky posts

  • if not used, the sticky posts will be treated as a normal post and return in chronological order

offset
integer (optional) Example: 1

Wordpress will ignore page post types if offset is used

page
integer (optional) Example: 1
per_page
integer (optional) Example: 3
order
string (optional) Example: DESC

Accepted values: ASC or DESC, default: DESC

orderby
string (optional) Example: title

Accepted values:

  • ID order by post ID

  • title order by title

  • type order by post type

  • date order by publish date

  • modified order by modified date

  • parent order by post parent

  • rand random order Default value: date
